# Structure H1 / H2 / H3 + titles & metas — Madame Wedding Design
### Basée sur l'analyse d'Easton Events, corrigée de leurs faiblesses
*À donner à Claude Code tel quel — c'est la carte des titres de TOUT le site. Juillet 2026.*

---

## 1. Ce qu'Easton Events nous apprend

**Ce qu'ils font bien (à reprendre)** : le H1 de leurs pages événement est **le lieu lui-même** (« Amalfi, Italy ») — le title devient « Amalfi, Italy Destination Wedding | Easton Events » : la destination porte le SEO, la sobriété porte le luxe. Leur meta description glisse « Ultra-Luxury », le nom du **photographe** (crédibilité presse + recherche croisée) et une seule phrase d'ambiance. Leur meta contact affirme la rareté : *« works with a select number of couples each year »*. Navigation minimale, maillage « Next Event » qui fait circuler.

**Leurs faiblesses (nos opportunités)** : hiérarchie de titres cassée sur la page contact (H6 et H5 *avant* le H1 — mauvais signal), pas de schema.org, quasi pas de texte sur les pages événement (les IA n'ont rien à citer !), et une privacy policy 100 % corporate qui rate le sujet que sa clientèle chérit : **la discrétion**. Madame fera l'inverse : hiérarchie parfaite, schema partout, un paragraphe citable par page, et une page « Privacy & Discretion » qui est un manifeste de marque autant qu'une page légale.

**Règles d'or pour tout le site** : un seul H1 par page ; ordre strict H1 → H2 → H3 (jamais de saut) ; le mot-clé principal dans le H1 et le title ; le title finit toujours par « | Madame Wedding Design » ; chaque page commence par un paragraphe de 1-2 phrases qui répond directement à l'intention de recherche (le format que les IA citent).

## 3. Consignes d'implémentation pour Claude Code

1. Un seul H1 par page, l'ordre H1→H2→H3 sans saut, jamais de heading utilisé « pour le style » (styler avec des classes, pas des balises).
2. Chaque page ouvre sur un paragraphe de réponse directe (1-2 phrases) sous le H1 — c'est ce que les moteurs génératifs citent.
3. Meta descriptions : 140–160 caractères, toujours avec un fait vérifiable (presse, classement, origine des couples) plutôt qu'un adjectif.
4. Ajouter le nom du **photographe** dans la meta et le schema des pages mariage publiées (l'astuce Easton : leurs noms sont cherchés et associent Madame à leur prestige).
5. Schema.org sur chaque gabarit (Organization, Article, FAQPage, BreadcrumbList) — l'avantage qu'Easton n'a pas.
