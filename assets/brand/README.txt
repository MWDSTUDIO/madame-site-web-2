assets/brand — Madame Wedding Design logo suite
================================================

Extracted programmatically from "Logo design.zip" (flat sand #DBD3C0 backgrounds
chroma-keyed to transparency, anti-aliased edges preserved, caption labels
removed). The embossed sand-on-sand cipher of 01-logo-principal cannot survive
background removal — it is recreated on the site as a CSS watermark layer
(cipher at 7% opacity behind the lettering: see .masthead .brand::before and
#intro .sas-logo::before in assets/css/site.css).

Files
  logo-espresso.png    primary logo (Madame script + WEDDING DESIGN caps) — default on ivory pages
  logo-hunter.png      primary logo, hunter — reserved for light print/stationery; never on hunter bg
  logo-ivory.png       generated declination — ONLY on the hunter entry sas / dark photography
  cipher-espresso.png  interlocked MWD monogram (watermark + footer)
  cipher-hunter.png    monogram, hunter — print contexts
  cipher-ivory.png     generated — threshold watermark
  avatar-espresso.png / avatar-hunter.png   disc avatars (social)
  lockup-espresso.png / lockup-hunter.png   compact two-line lockups (reserved)
  favicon/             favicon-512/192/32/16.png, apple-touch-icon.png (180, solid ivory), favicon.ico
  reference/           00-suite-complete presentation boards — NEVER use on the site
  seal.svg             earlier line-work seal, superseded by the logo suite (kept for reference)
  logo.svg             earlier wordmark placeholder, superseded (kept for schema fallback history)

Rules
  - Espresso on ivory; ivory on hunter/dark. Never hunter on hunter, never espresso on hunter.
  - Logos always sit on transparency — never boxed in a sand rectangle.
